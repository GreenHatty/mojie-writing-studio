# 墨界·私人网文创作台

墨界是面向受邀中文网文作者的创作、设定、云同步、备份和发布准备平台。网站采用“本机即时草稿 + 服务端权限与云端修订”的双层架构，作者始终保留对正文修改和正式发布的最终控制权。

## 已实现

### 身份与数据安全

- 首次站点所有者初始化；
- 72小时一次性邀请注册；
- PBKDF2-SHA256 密码派生与独立盐值；
- HttpOnly、Secure、SameSite=Strict 会话；
- `owner / admin / writer / editor / commenter / viewer` 六级角色；
- 服务端按会话和作品成员关系重新校验每次读写；
- 每位用户独立的本机 IndexedDB 数据空间；
- D1 云端作品、修订历史、409版本冲突保护和审计日志。

### 写作与设定

- 多作品工作台：长篇、短篇与随笔；
- 桌面三栏、平板双栏、手机抽屉式编辑器；
- Tiptap 富文本、章节目录、自动保存、离线草稿、冲突副本和命名版本；
- 大纲、人物、地点、时间线、人物关系、世界观、势力、素材、目标与词典；
- 时间冲突检查、人物关系图、地图DIY与SVG导出；
- 全书普通/正则搜索、上下文预览、替换前快照和HTML保真替换；
- 起点、番茄与通用方向的男频、女频和短故事模板；
- 原创网文基础课程、随机取名、快捷短语和小说数值计算器；
- 标点、成对符号、风险词、重复词、重复片段和长句检查；
- 章节节奏、对话占比、结尾钩子、场景八拍、简介生成、保守精简与伏笔候选；
- 时间/字数双目标写作冲刺和安全退出。

### DOCX

- 读取标准 DOCX OOXML/ZIP 包；
- 未修改时按原始字节导出，SHA-256 与导入文件一致；
- 原格式编辑模式只替换现有正文文本节点；
- 未编辑的图片、样式、编号、关系、页眉页脚、脚注等包部件原样保留；
- 原格式模式要求段落数量不变，超出保证范围时拒绝导出而非虚假承诺；
- 原件和编辑件可分别存入权限隔离的 R2 对象存储。

### 排行榜与发布

- 配置起点、番茄各榜单与分类的已授权官方来源网址；
- Worker Cron 自动采集每个来源前十名公开元数据；
- 保存每日快照、来源哈希、成功时间和错误记录；
- 统计前十名共同热点元素、机制承诺、身份承诺和冲突承诺；
- 不抓取付费章节正文，不绕过登录、验证码或访问限制；
- 起点、番茄作者后台直达，以及纵横、17K、晋江作者入口；
- 逐章发布前检查、标题/正文复制和作者人工最终确认。

### 云同步与自动备份

- 作品云端修订同步和冲突拒绝覆盖；
- R2、WebDAV、S3兼容对象存储；
- 用户自定义自动备份间隔：5分钟至30天；
- 用户自定义临时备份保留期限：1小时至365天；
- AES-GCM 加密保存 WebDAV/S3 凭据；
- Cron 自动创建到期备份并删除过期对象；
- 关闭策略后，旧临时备份仍按其原到期时间清理。

## 本地运行

需要 Node.js 22。Vinext 与 Cloudflare 适配器使用预发布版本：

```bash
npm ci --legacy-peer-deps
npm run dev
```

质量检查：

```bash
npm test
npm run typecheck
npm run build
npm run test:worker-entry
```

## 生产部署

完整部署步骤见 [`docs/cloud-deployment.md`](docs/cloud-deployment.md)，包括：

- Cloudflare D1 数据库；
- DOCX 与备份 R2 Bucket；
- `migrations/0001_cloud.sql` 数据库迁移；
- `MOJIE_ADMIN_TOKEN` 首次初始化密钥；
- `MOJIE_BACKUP_MASTER_KEY` 备份凭据加密密钥；
- Worker Cron 和环境变量。

没有绑定 D1 时，网站会明确显示服务端尚未配置，不会退化为伪造的多用户权限系统。数据库、存储桶和托管密钥必须由仓库所有者在部署平台创建；它们不能安全地写入代码仓库。

## 隐私与凭据

- 本地正文辅助默认不调用外部模型；
- 平台账号密码不会保存到墨界；
- WebDAV/S3凭据只以服务端密文保存；
- 服务端密钥不得使用 `NEXT_PUBLIC_` 前缀；
- 发布、批量替换、DOCX编辑和智能建议均保留人工确认或版本回退能力。

本分支保持草稿 PR，不自动合并 `main`，也不在未配置生产资源时声称网站已经正式上线。
