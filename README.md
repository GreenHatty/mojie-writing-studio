# 墨界·私人网文创作台

墨界是面向受邀中文网文作者的创作、设定、有限协作、云同步、备份和发布准备平台。网站采用“本机即时草稿 + 服务端权限与云端修订”的双层架构，作者始终保留对正文修改、修改建议和正式发布的最终控制权。

当前部署模式为：**Cloudflare D1 云端正文 + 浏览器本机 DOCX 文件库 + 用户自选 WebDAV/S3 备份**。项目不启用 Cloudflare R2，也不会自动创建 R2 资源。

## 已实现

### 身份、权限与隐私

- 首次站点所有者初始化；
- 账户级邀请与作品级一次性邀请；
- 带版本标识的 PBKDF2-SHA256 密码派生、100,000 次迭代与独立随机盐；
- 登录失败执行恒定密码派生，降低账号枚举差异；
- HttpOnly、Secure、SameSite=Strict 会话；
- `owner / admin / writer / editor / commenter / viewer` 六级角色；
- 所有私人内容接口都要求明确的 `work_members` 成员关系，全局管理员不会自动获得正文访问权；
- 作品成员列表、直接授权、撤销权限和会话失效；
- 每位用户独立的本机 IndexedDB 数据空间；
- D1 云端作品、修订历史、409版本冲突保护和审计日志。

### 批注、建议与管理后台

- 章节级和选区级批注，批注不会修改正文；
- 原文/建议双栏对比；
- 接受建议前再次核对章节、位置和原文，锚点失效时拒绝自动替换；
- 接受、拒绝、重新打开和处理状态；
- Owner/Admin 运行概览、用户状态、邀请记录和审计日志；
- Owner 可配置站点名称、默认邀请时长和回收站默认保留期；
- 管理后台不提供任意浏览其他作者正文的普通入口。

### 写作与设定

- 多作品工作台：长篇、短篇与随笔；
- 桌面三栏、平板双栏、手机抽屉式编辑器；
- Tiptap 富文本、章节目录、自动保存、离线草稿、冲突副本和命名版本；
- 结构化章节细纲：状态、目标字数、视角、时间、地点、人物、冲突、信息增量、情绪、伏笔和结尾钩子；
- 大纲、人物、地点、时间线、人物关系、世界观、势力、素材、目标与词典；
- 时间冲突检查、人物关系图、地图DIY与SVG导出；
- 全书普通/正则搜索、上下文预览、替换前快照和HTML保真替换；
- 起点、番茄与通用方向的男频、女频和短故事模板；
- 原创网文基础课程、随机取名、快捷短语和小说数值计算器；
- 标点、成对符号、风险词、重复词、重复片段和长句检查；
- 超过阈值的文本检查通过 Web Worker 执行，避免阻塞输入；
- 章节节奏、对话占比、结尾钩子、场景八拍、简介生成、保守精简与伏笔候选；
- 时间/字数双目标写作冲刺和安全退出。

### 离线与 PWA

- 可安装的 Web App Manifest；
- Service Worker 只缓存同源静态资源；
- `/api/`、页面导航和私人 HTML 明确不进入离线缓存；
- 网络中断时当前章节继续使用 IndexedDB 草稿；
- 恢复网络后仍通过版本号进行安全同步。

### DOCX

- 读取标准 DOCX OOXML/ZIP 包；
- 未修改时按原始字节导出，SHA-256 与导入文件一致；
- 原格式编辑模式只替换现有正文文本节点；
- 未编辑的图片、样式、编号、关系、页眉页脚、脚注等包部件原样保留；
- 原格式模式要求段落数量不变，超出保证范围时拒绝导出而非虚假承诺；
- 原件和编辑件按当前账号及作品保存在浏览器 IndexedDB；
- 支持重新载入、下载和删除本机 DOCX；
- DOCX 不上传 Cloudflare R2，因此不会产生 R2 存储费用。

### 排行榜与发布

- 配置起点、番茄各榜单与分类的已授权官方来源网址；
- Worker Cron 自动采集每个来源前十名公开元数据；
- 保存每日快照、来源哈希、成功时间和错误记录；
- 统计前十名共同热点元素、机制承诺、身份承诺和冲突承诺；
- 不抓取付费章节正文，不绕过登录、验证码或访问限制；
- 起点、番茄作者后台直达，以及纵横、17K、晋江作者入口；
- 逐章发布前检查、标题/正文复制和作者人工最终确认。

### 云同步与自动备份

- D1 作品云端修订同步和冲突拒绝覆盖；
- 自动备份目标为用户自己的 WebDAV 或 S3兼容对象存储；
- 用户自定义自动备份间隔：5分钟至30天；
- 用户自定义临时备份保留期限：1小时至365天；
- AES-GCM 加密保存 WebDAV/S3 凭据；
- Cron 自动创建到期备份并删除过期对象；
- 关闭策略后，旧临时备份仍按其原到期时间清理；
- 未配置第三方存储时不会伪装成已完成远程备份。

## 本地运行

需要 Node.js 22。Vinext 与 Cloudflare 适配器使用预发布版本：

```bash
npm ci --legacy-peer-deps
npm run dev
```

质量检查：

```bash
node scripts/test-privacy-guard.mjs
npm test
npm run typecheck
npm run build
npm run test:worker-entry
```

## 生产部署

完整部署步骤见 [`docs/cloud-deployment.md`](docs/cloud-deployment.md)，包括：

- Cloudflare D1 数据库；
- `migrations/0001_cloud.sql` 与 `migrations/0002_collaboration_admin.sql`；
- `MOJIE_ADMIN_TOKEN` 首次初始化密钥；
- `MOJIE_BACKUP_MASTER_KEY` 备份凭据加密密钥；
- Worker Cron 和环境变量；
- 可选 WebDAV 或 S3 兼容存储。

没有绑定 D1 时，网站会明确显示服务端尚未配置，不会退化为伪造的多用户权限系统。D1 和托管密钥必须由仓库所有者在部署平台创建；它们不能安全地写入代码仓库。R2 不属于当前部署前置条件。

## 隐私与凭据

- 本地正文辅助默认不调用外部模型；
- 平台账号密码不会保存到墨界；
- WebDAV/S3凭据只以服务端密文保存；
- 服务端密钥不得使用 `NEXT_PUBLIC_` 前缀；
- Service Worker 不缓存认证 API 或私人页面；
- 发布、批量替换、DOCX编辑和智能建议均保留人工确认或版本回退能力。

本分支保持草稿 PR，不自动合并 `main`，也不自动进行正式生产部署。
