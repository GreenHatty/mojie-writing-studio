# 墨界·私人网文创作台

墨界是一个面向中文网文作者的离线优先创作台。现有版本以正文安全、中文写作体验和可恢复性为优先目标，保留作者对所有修改的最终控制权。

## 已实现

- 多作品工作台：长篇、短篇与随笔项目；
- 桌面三栏编辑器，平板双栏、手机抽屉式布局；
- Tiptap 富文本编辑、章节目录、即时字数、阅读时长、引用、分隔线、标题和格式清理；
- IndexedDB 自动保存、离线草稿、修订冲突副本、命名版本与恢复前快照；
- 本章备注，导出时自动排除私人备注；
- 大纲、人物、地点、时间线、关系、世界观、势力、素材、目标与词典记录；
- 时间冲突检查、人物关系图、轻量地图DIY、路线保存与SVG导出；
- 软删除与设定回收站；
- 全书查找、大小写/正则搜索、上下文预览、替换前版本与HTML保真替换；
- 起点、番茄和通用方向的男频、女频及短故事结构模板；
- 原创网文基础课程；
- 标点、成对符号、风险词、重复词、重复片段、长句与半角标点检查；
- 现代姓名、古代姓名、宗门、地点、功法、武器、丹药、组织和科幻代号生成；
- 快捷短语、编辑器直接插入、时间/字数双目标写作冲刺和安全退出；
- 基础公式、年龄、旅程、复合增长和多次概率计算器；
- TXT、Markdown 和墨界项目包预览导入；
- TXT、Markdown、HTML 和墨界 JSON 项目备份导出；
- 手动导入起点/番茄公开榜单元数据、每日前10快照与结构化卖点拆解；
- 逐章发布前检查、干净副本、标题/正文复制与人工确认；
- 纸白、暖黄、低对比灰、深色主题，字号、行距与编辑宽度调整；
- GitHub Actions 质量门禁配置。

## 运行

需要 Node.js 22。Vinext 和其 Cloudflare 适配器目前使用预发布版本，npm 需要采用兼容 peer 解析方式：

```bash
npm ci --legacy-peer-deps
npm run dev
```

质量检查：

```bash
npm test
npm run typecheck
npm run build
npm run test:worker-entry
```

## 数据与隐私

默认数据保存在当前浏览器的 IndexedDB 或仅用于偏好类数据的 Local Storage 中。该模式适合个人写作和受控设备，但**不等同于真正的邀请制多用户安全系统**。正式向受邀用户开放前，需要按 [`docs/server-integration.md`](docs/server-integration.md) 接入服务端身份验证、权限校验、云端数据库和对象存储。

正文检查、模板和课程均在本地运行。未配置智能服务时，正文不会发送给外部模型。

## 导入与DOCX边界

TXT、Markdown 和墨界 JSON 项目包支持预览后导入。DOCX 的可靠往返需要正式文档解析器和对象存储：导入时必须保留原始文件，复杂文本框、域、页眉页脚、批注和分节不能虚假承诺完全无损。仓库不会把 HTML 改扩展名伪装成 DOCX。

## 排行榜与发布边界

排行榜首版只接受管理员手动导入的合法公开元数据，不抓取付费正文，不绕过验证码或反爬机制。卖点拆解会标记为基于书名、标签和简介的结构性推测。

仓库不会保存起点或番茄账号密码，不处理验证码，也不调用未经授权的私有发布接口。当前发布功能只生成发布副本、检查格式和风险，并要求作者在平台后台亲自确认。作者后台地址必须通过公开环境变量配置：

```text
NEXT_PUBLIC_QIDIAN_AUTHOR_URL
NEXT_PUBLIC_FANQIE_AUTHOR_URL
```

## 部署

项目包含 Vinext/Vite 与 Cloudflare Worker/Sites 兼容配置。提交部署前必须完成全部质量命令，并确认环境变量中没有将数据库密钥、存储密钥、平台令牌或模型密钥暴露为 `NEXT_PUBLIC_*`。

本分支只用于审查，不自动合并 `main`，也不自动公开部署。
