# 架构

墨界采用浏览器功能、受保护 HTTP API、应用服务和运行时适配器四层结构。界面组件只能调用 API 或客户端服务，不能直接访问 Worker binding、环境变量或他人数据。

生产运行时通过 Cloudflare Worker binding 提供 D1 `DB` 和 R2 `OBJECTS`。关系数据由服务端数据库适配器保存；封面、原始导入、导出、备份和附件由对象存储适配器保存。浏览器 IndexedDB 仅用于当前用户加密草稿、同步队列和偏好，不能替代服务端私有数据。

每个受保护请求按以下顺序处理：会话验证、变更请求的 CSRF 验证、输入验证、作品权限验证、事务性业务服务、无正文审计。正文标准格式为 Tiptap JSON；纯文本只由该 JSON 派生。
