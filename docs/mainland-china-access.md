# 中国大陆直连方案与前置条件

更新时间：2026-07-14

## 当前结论

当前生产地址使用 `workers.dev`。它可以继续作为全球预览和灾备入口，但无法由项目代码保证在中国大陆稳定直连。代码侧已经做到：不依赖 Google Fonts、海外 CDN 或第三方前端脚本；字体、地图图片、编辑器和静态资源均随 Worker Assets 自托管。

要把“可直接访问”变成可以验收的生产承诺，必须先具备域名、备案和境内网络服务权限。任何只改 DNS 或把 `workers.dev` 换一个别名的做法，都不能代替这些条件。

## 官方 Cloudflare 路径

1. 准备自有顶级域名，并完成适用于该域名的 ICP 备案或许可证。
2. 使用 Cloudflare Enterprise，并单独订购 Cloudflare China Network。
3. 将域名、ICP 信息、内容说明和自我声明提交给 Cloudflare 的中国合作方京东云进行内容审核。
4. 启用中国境内权威 DNS / China Network；在网页页脚展示备案号。
5. 本站依赖 Workers、Assets、Secrets 和 D1。Cloudflare 当前中国网络产品清单明确列出 Workers、Assets 和 Secrets，但没有把 D1 列为境内可用产品，因此上线前必须由 Cloudflare 客户团队书面确认 D1 绑定的运行位置和可用性。
6. 如果 D1 仍在境外，需评估 Cloudflare Global Acceleration，或把统一仓储层接到已备案的中国大陆数据库服务。不得为了上线复制出第二套权限逻辑。

## 可选中国大陆镜像路径

如果不采用 Cloudflare Enterprise China Network，可在取得域名和 ICP 后，把同一版本部署到中国大陆云厂商的边缘函数 / 容器服务，并为统一仓储层实现经过测试的数据库适配器。该路径涉及新的云账户、备案主体、域名、密钥和数据库授权，不能由代码仓库单独完成。

## 上线验收

- 在北京、上海、广州、成都至少四个网络探针验证 DNS、TLS、首屏、登录、D1 读写、图片资源和断线重连。
- 验证所有私有 API 继续返回 `Cache-Control: no-store`，中国 CDN 不缓存正文、Cookie、令牌或私人 HTML。
- 验证地图资源从站点自有域名加载，核心设定优化器只连接用户选择且被服务器允许的模型域名。
- 验证备案号、隐私说明和第三方模型数据传输说明已展示。
- 保留全球生产入口作为回滚目标；境内配置异常时不得覆盖或删除正式 D1 数据。

## 当前需要用户提供

- 自有域名。
- ICP 备案号 / 许可证及备案主体。
- Cloudflare Enterprise China Network 或中国大陆云厂商账号授权。
- 选定的境内数据架构，以及服务商对 D1 / 外部数据库连接的书面可用性确认。
