# PR B 私人写作核心验收记录

## 范围

本阶段把站点首页切换到安全基础架构的核心认证与数据路径，完成：

- Owner/Writer 平台身份登录、会话恢复与每用户本地草稿密钥解封。
- 作品工作台，长篇、短篇、随笔创建与今日字数、连续写作统计。
- 卷与章目录、分卷创建/重命名、章节创建/重命名/排序/切换。
- Tiptap JSON 正文、实时字数、撤销重做、专注模式与四套主题。
- 输入立即写入每用户加密 IndexedDB，约一秒后按 `baseRevision` 和 `clientOperationId` 幂等同步。
- 断网写作、重新联网自动入队同步、刷新后恢复未同步草稿。
- 冲突草稿停止自动重试，同时保留本地草稿与云端冲突版本；继续编辑或恢复版本均不会静默覆盖。
- 私人章节备注、命名版本、恢复前快照、全书搜索、章节回收站。
- 桌面三栏、1024px 平板上下文抽屉和 390px 手机目录/上下文抽屉。
- 个人外观偏好同时写入本地加密存储与 `profile_settings`；失败时加密排队，联网后重试。

## 数据与安全边界

- 所有核心 API 复用统一会话摘要、CSRF 与服务端作品权限判断。
- 作品列表只返回当前用户拥有或被授权的作品；平台 Owner 身份不扩大作品正文权限。
- 正文标准格式为带 `schemaVersion` 的 Tiptap JSON，`plain_text` 仅由服务端派生。
- 所有受保护响应使用 `Cache-Control: no-store, private`。
- 浏览器只保存加密草稿、同步操作、冲突副本和偏好；DEK 只存在于当前内存会话。
- 删除章节仅写入软删除字段；最后一个活动章节不能被删除。
- 未新增 R2 Bucket、远程 D1、生产密钥或任何部署资源。

## 自动验收

通过以下验证：

- `npm run typecheck`
- `npm test`：60 个测试文件、151 条测试。
- `npm run build`
- `npm run test:e2e`：桌面、1024px 平板、390px 手机共 6 条流程。
- `npm run test:core-worker-local`：本地临时 D1 + Worker 覆盖认证、草稿密钥、设置、目录、备注、版本、搜索、回收站、统计、幂等保存、冲突与会话撤销。
- `node scripts/test-privacy-guard.mjs`
- `npm run test:worker-entry`
- `npm run test:d1-local`

真实浏览器额外验证了登录、打开作品、正文输入、自动保存、刷新保留、断网编辑、重连同步、平板抽屉和 390px 手机布局。该过程发现并修复了“离线草稿未入队时重连不自动同步”的缺陷。
