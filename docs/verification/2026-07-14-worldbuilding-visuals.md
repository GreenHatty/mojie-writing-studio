# PR D 大纲、设定与可视化验收记录

## 交付范围

- 作品级 `project_entities` 关系表及受保护 API；接口参数统一使用 `workId`。
- 故事总纲、主线、副线、分卷大纲、伏笔线、章节细纲和场景目标。
- 人物、别名、地点、势力、世界规则与素材卡；素材界面要求保存用户自己的摘要和来源记录。
- 时间线事件关联人物、地点、章节、前置事件和伏笔标记。
- 人物出生/死亡边界、事件起止、前置顺序和同人物重叠地点冲突提示。
- 人物/势力关系图、地图地点、区域、路线、图层、坐标与 SVG 本地导出。
- 正文右栏按需提示当前章节出现的人物、别名、地点和势力；临时 Decoration 高亮不写入 Tiptap 正文。
- 设定软删除、回收站恢复与删除前引用清单，不执行级联删除。

## 权限与数据边界

- 平台 Owner 身份不扩大作品权限；无作品访问权的 Writer 得到 404，不暴露实体是否存在。
- Viewer/Commenter 只能读取已有访问范围内的设定；只有作品拥有者和 Editor 可创建、修改、删除或恢复。
- 所有响应使用 `Cache-Control: no-store, private`，所有写入继续要求有效会话、同源校验和 CSRF。
- `fields_json` 只接受受限的字符串、有限数字、布尔值、字符串数组和空值，并限制请求体、字段数、数组长度与单字段长度。
- 可视化模块动态导入；工作台、作品打开和正文输入不会预加载设定、图谱或地图数据。
- 没有新增 R2、生产 D1、对象存储 Bucket、密钥或部署资源。

## 验收结果

- `npm run typecheck`
- `npm test`：62 个测试文件、162 条测试。
- `npm run build`
- `npm run test:d1-local`：包含 `project_entities` 迁移。
- `npm run test:core-worker-local`：覆盖无权限隐藏、Viewer 只读、Editor 可写、引用确认、软删除与恢复。
- `npm run test:e2e`：8 条通过、4 条按设备条件跳过；覆盖桌面、1024px 平板和 390px 手机核心流程，桌面额外覆盖人物创建、正文命中和非持久化高亮。
- E2E 使用生产构建与 `vinext start`，避免开发服务器依赖预打包重载中断后续设备流程。
- 响应式专项验证了平板和 390px 手机可在两层内打开并关闭“大纲与设定”，手机抽屉宽度不超过视口。
