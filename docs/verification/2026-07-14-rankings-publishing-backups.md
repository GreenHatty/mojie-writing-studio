# PR E 验收记录：榜单、发布与外部备份

## 交付边界

- 新增独立的 `core_ranking_*`、`core_publication_records`、`backup_targets`、`backup_runs` 与 `backup_objects_v2` 表。
- 不读取或写入旧 `cloud_documents.payload_json` 总包；旧表与旧正式数据保持不变。
- 榜单仅在用户打开“平台榜单”标签后读取来源摘要和每个来源最新成功快照。
- 榜单采集请求只创建任务并返回 `202`；Worker 使用 `waitUntil` 执行短任务，Cron 继续处理排队任务。
- 手动导入仅接收最多 2 MB 的脱敏 CSV/JSON 公开元数据；空结果、验证码、解析失败和非法跳转不会覆盖成功快照。
- 发布功能只做本地检查、复制、打开作者后台与人工发布记录，不保存平台密码、不自动登录、不处理验证码、不自动发布。
- 外部备份直接从规范化作品、卷、章、版本、私人备注和设定表生成；WebDAV/S3 凭据使用 AES-GCM 加密后保存。
- R2 备份绑定从构建配置中移除；没有创建 Bucket，也不作为开发、测试或部署前置条件。

## 安全边界

- 所有新接口使用平台基础会话摘要、同源检查和双提交 CSRF。
- 平台 `OWNER` 只拥有榜单来源管理权，不会自动获得其他作者作品正文、发布记录或备份的读取权。
- 发布记录要求作品拥有者或 `EDITOR`；外部备份目标仅允许作品拥有者创建。
- WebDAV/S3 地址必须是 HTTPS，拒绝 URL 内凭据、localhost、环回与常见私网地址，并拒绝重定向。
- 下载外部备份时限制响应体大小并验证 SHA-256 完整性。
- `MOJIE_BACKUP_MASTER_KEY` 缺失时失败关闭，前端只显示“已实现但需外部配置”，不降级为明文。
- 日志只记录非敏感错误码，不记录正文、Cookie、令牌或外部存储凭据。
