# PR C 模板、检查、课堂与文件验收记录

## 范围

本阶段在私人写作核心之上增加按需加载的“写作工具箱”，完成：

- 起点、番茄与通用平台的五层题材模板，支持筛选并生成可编辑策划卡。
- 策划卡、个人敏感词、白名单、过度使用词和快捷词均写入当前用户的加密 IndexedDB 命名空间。
- 非破坏性错别字、标点、敏感词、重复短语与基础风格检查；结果只提示，不直接替换正文。
- 随机取名、快捷词、数值计算、小黑屋写作冲刺与原创写作课堂。
- TXT、Markdown、HTML、schema v2 JSON 项目包、受限 ZIP 项目包和基础 DOCX 的导入导出。
- 导入先预览卷章数量，再选择加入当前作品或创建新作品；每章独立写入，不覆盖已有正文。
- DOCX 原始文件使用每用户独立的浏览器文件库保存；复杂 Word 样式只提供原文件保留与支持范围内的基础往返，不宣称完全无损。

## 安全与性能边界

- 工具箱使用动态导入和局部错误边界；失败不会卸载编辑器或阻止正文保存、切章与返回工作台。
- 长文本重复短语分析移至 Web Worker，交互主线程只对较短文本执行有上限的即时分析。
- HTML 导入移除脚本、样式、iframe 和 object；服务端再次规范化所有导入的 Tiptap JSON，只保留允许的节点、标记与安全属性。
- JSON/ZIP 导入校验 schemaVersion、文件体积、解压后体积、条目数、分卷数、章节数和单章字符数。
- 导出文件不包含私人备注、会话、Cookie、令牌、草稿密钥或登录信息。
- DOCX 浏览器文件库按 userId 命名空间隔离；没有新增 R2 Bucket、生产密钥、远程数据库或部署资源。
