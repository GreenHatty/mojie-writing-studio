# 性能预算与测量方案

## 不可回归预算

| 场景 | 预算 |
| --- | --- |
| 正文输入主线程任务 | 目标低于 50ms，任何辅助模块不得产生超过 100ms 的阻塞任务 |
| 启动进入可操作或恢复状态 | 本地正常路径 3 秒内；所有远端请求 12 秒超时后必须进入可恢复终态 |
| 创建作品与保存 | 15 秒超时；本地持久化成功后云端失败不得阻止继续写作 |
| 切换章节 | 当前章本地 flush 完成即可切换；云端失败进入幂等队列 |
| 工作台 | 不加载正文、榜单历史、图谱、DOCX 或管理后台模块 |
| 排行榜 | 用户主动打开后加载；只取来源和最新快照；抓取通过 202 后台任务 |
| 大型目录和搜索 | 虚拟化或分页，不一次渲染全书正文 |

## 测量场景

- 1440×900、1024×768、390×844 三视口。
- 首次启动、登录、创建作品、打开作品、连续中文输入、离线追加、切章、返回工作台。
- IndexedDB blocked、versionchange、升级失败、多标签占用及只读恢复。
- 打开排行榜、图谱、DOCX、备份和管理后台并注入失败。
- 五百万中文字符等价的分章测试数据，只按需加载当前章。

## 工具与证据

- Playwright E2E 记录流程、视口、水平溢出、错误终态和 Long Task。
- Vitest 对状态机、幂等队列和纯解析器进行确定性测试。
- Chrome/Edge Performance 记录输入、切章和辅助模块动态加载。
- Cloudflare 临时 D1 验证真实路由、迁移和后台任务；结束后删除临时资源。
- 每个 PR 更新验证报告，缺少外部账号的能力只能标记“已实现但需外部配置”。

## 停止条件

出现无限 loading、按钮无法解锁、正文输入超过 100ms 长任务、辅助模块阻止保存/切章、空榜单覆盖成功快照或 Service Worker 缓存私人内容时，立即停止后续堆叠 PR，先修复回归。
