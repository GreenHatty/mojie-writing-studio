## 已验证的核心行为

- 首次用户能够创建作品并进入第一章；
- 多作品工作台能够列出并打开指定作品；
- 章节、目录和设定数据在 IndexedDB 中持久化；
- 本地草稿不会直接覆盖已同步修订；
- 过期修订保存会建立冲突副本；
- 命名版本可建立和恢复，恢复前自动留存快照；
- 项目实体支持创建、查询、软删除和恢复；
- 文本检查只返回问题范围和级别，不直接修改正文；
- 随机取名在给定种子时结果可重复；
- 模板可按平台、读者、篇幅、题材和元素过滤；
- TXT/Markdown章节拆分、项目包往返和HTML安全转义通过测试；
- 发布准备不会保存密码或标记为自动发布；
- 安全公式、年龄、旅程、复合增长和概率计算通过测试；
- 时间线可检测同一人物在重叠时间出现于不同地点；
- 地图与关系图SVG导出会转义用户标签；
- 全文查找支持普通文本和正则；
- 全文替换不会改动HTML标签属性；
- 排行榜CSV/JSON导入只保留每组前10名；
- 卖点分析只使用公开书名、标签和简介，并附结构性推测声明；
- Cloudflare/Sites Worker入口文件构建后存在且可加载。

## 不能在无外部条件时虚假宣称完成的项目

以下能力依赖站点所有者提供正式后端、密钥、对象存储或平台授权。仓库已提供能力开关、数据契约和接入说明，但默认不会伪造成功状态：

1. **真正的邀请制多用户认证和服务端权限隔离**：需要身份提供方、服务端会话和云数据库；
2. **跨设备实时云同步和多人同时编辑**：需要云端版本服务或协作协议；
3. **复杂DOCX无损往返**：需要正式DOCX解析器和对象存储，并保留原始文件；
4. **合法自动更新平台榜单**：需要官方公开接口、正式授权或外部合规任务；
5. **起点/番茄全自动发布**：必须取得官方作者写入接口授权；
6. **AI辅助写作**：需要站点所有者选择合规模型服务、配置密钥和隐私协议；
7. **WebDAV或外部对象存储备份**：需要外部服务地址和用户授权。

这些边界详见 `docs/server-integration.md`。未配置时，界面会明确显示本地优先模式，不保存第三方平台密码，不绕过验证码，不抓取付费正文，也不将正文发送给外部模型。

## 发布建议

- 保持 PR 为草稿，先由所有者在预览环境进行真实浏览器验收；
- 重点测试中文输入法、长章节、断网恢复、手机抽屉、导入真实作品副本和批量替换；
- 配置真正的认证和云存储前，不应向不受信任的外部用户公开；
- 所有服务端密钥只放入托管密钥或环境变量，不得使用 `NEXT_PUBLIC_` 前缀；
- 通过预览验收后再合并和部署，不直接在本次改动中自动上线。
