## 已验证的核心行为

### 本机创作

- 首次用户能够创建作品并进入第一章；
- 多作品工作台能够列出并打开指定作品；
- 章节、目录和设定数据在用户独立的 IndexedDB 中持久化；
- 本地草稿不会直接覆盖已同步修订；
- 过期修订保存会建立冲突副本；
- 命名版本可建立和恢复，恢复前自动留存快照；
- 项目实体支持创建、查询、软删除和恢复；
- 全文查找支持普通文本和正则；
- 全文替换不会改动HTML标签属性；
- 时间线可检测同一人物在重叠时间出现于不同地点；
- 地图与关系图SVG导出会转义用户标签。

### 写作辅助

- 文本检查只返回问题范围和级别，不直接修改正文；
- 随机取名在给定种子时结果可重复；
- 模板可按平台、读者、篇幅、题材和元素过滤；
- 安全公式、年龄、旅程、复合增长和概率计算通过测试；
- 章节节奏、对话比例、结尾钩子和场景八拍输出稳定；
- 简介生成只使用用户输入的主角、身份、目标、机制和代价；
- 保守精简不会自动覆盖正文。

### DOCX

- 标准 OOXML ZIP 包可以导入；
- 未修改时导出字节与原文件完全一致，SHA-256相同；
- 修改正文后，包内样式、页眉、媒体和其他未编辑部件保留；
- 混合粗体/斜体运行节点在文字替换后继续保留；
- 原格式模式会拒绝段落数量变化，避免超出保证范围后伪称无损；
- 浏览器二进制接口使用明确的 `ArrayBuffer`，通过严格TypeScript检查。

### 服务端与自动任务

- Worker API、构建脚本和入口脚本通过 Node 语法检查；
- D1迁移定义用户、邀请、会话、成员、作品修订、DOCX资产、榜单、备份和审计表；
- 角色权限矩阵和备份间隔/保留期限校验通过测试；
- Worker构建产物包含认证API、榜单执行API、备份执行API和Scheduled入口；
- 生成的Worker配置包含Cron触发器；
- 生产构建和Sites Worker入口校验通过。

## 已完成的功能范围

- 受邀登录、首次所有者初始化、一次性邀请和六级角色模型；
- 服务端会话与作品成员权限隔离；
- 云端作品修订、冲突保护和审计日志；
- DOCX原件与原格式编辑件导入、导出及R2存储；
- 授权起点/番茄榜单来源、定时前十采集和热点共性分析；
- 起点、番茄及其他网文平台作者后台直达；
- 多功能章节规划、节奏分析、钩子、简介、精简和伏笔辅助；
- R2、WebDAV、S3兼容存储自动临时备份；
- 用户自定义备份间隔和自动删除期限；
- 过期备份对象定时删除。

## 仍需部署者提供的外部资源

代码和界面已经完成，但以下资源不能安全地由仓库代码自动创建或填入：

1. Cloudflare D1 数据库ID；
2. DOCX与备份R2 Bucket；
3. `MOJIE_ADMIN_TOKEN`；
4. `MOJIE_BACKUP_MASTER_KEY`；
5. 实际WebDAV或S3兼容存储账号；
6. 用户所述的起点/番茄抓取授权来源网址；
7. 生产域名和部署权限。

完整配置步骤见 `docs/cloud-deployment.md`。未绑定D1时，登录页明确显示服务端尚未配置，不会退化成假身份验证；没有对象存储或密钥时，对应接口返回配置错误，不会声称备份或DOCX云存储成功。

## DOCX保证范围

- **完全字节无损：** 导入后未修改，直接导出原件；
- **格式结构保留编辑：** 段落数量不变时，保留原OOXML包和运行格式节点，仅替换正文文字；
- **不作虚假保证：** 任意增删段落、重排复杂文本框或重构Word对象时，不宣称与原文件逐字节相同。
