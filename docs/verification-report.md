## 自动化质量门禁

`Quality` 工作流依次执行：

1. `npm ci --legacy-peer-deps --loglevel=error --no-audit --no-fund`
2. Node 语法检查：
   - `scripts/mojie-api.mjs`
   - `scripts/mojie-extended-api.mjs`
   - `scripts/mojie-privacy-guard.mjs`
   - `scripts/cloudflare-fetch-entry.mjs`
   - `scripts/prepare-dist.mjs`
   - `public/sw.js`
   - `public/text-check-worker.js`
3. `npm test`
4. `npm run typecheck`
5. `npm run build`
6. `npm run test:worker-entry`

失败时分别上传单元测试、TypeScript 或生产构建诊断，不允许通过跳过步骤获得绿色状态。

## 已自动验证的核心行为

### 本机创作与写作辅助

- 首次用户能够创建作品并进入第一章；
- 多作品工作台能够列出并打开指定作品；
- 章节、目录和设定数据在用户独立的 IndexedDB 中持久化；
- 本地草稿不会直接覆盖已同步修订；
- 过期修订保存会建立冲突副本；
- 命名版本可建立和恢复，恢复前自动留存快照；
- 项目实体支持创建、查询、软删除和恢复；
- 全文查找支持普通文本和正则；
- 全文替换不会改动HTML标签属性；
- 时间线可检测同一人物在重叠时间出现于不同地点；
- 地图与关系图SVG导出会转义用户标签；
- 文本检查只返回问题范围和级别，不直接修改正文；
- 随机取名在给定种子时结果可重复；
- 模板可按平台、读者、篇幅、题材和元素过滤；
- 安全公式、年龄、旅程、复合增长和概率计算通过测试；
- 章节节奏、对话比例、结尾钩子和场景八拍输出稳定；
- 简介生成只使用用户输入的主角、身份、目标、机制和代价；
- 保守精简不会自动覆盖正文。

### 批注与修改建议

- 选区范围会规范化，不接受负数或倒置位置；
- 修改建议只有在当前位置的文字仍与建议原文一致时才允许应用；
- 原文已变化时返回锚点失效，不把旧建议套用到错误位置；
- 相同原文与替换文本不会被视为有效修改；
- 批注、建议和接受状态通过服务端作品权限控制；
- 接受成功后继续进入普通自动保存与版本冲突流程。

### DOCX

- 标准 OOXML ZIP 包可以导入；
- 未修改时导出字节与原文件完全一致，SHA-256相同；
- 修改正文后，包内样式、页眉、媒体和其他未编辑部件保留；
- 混合粗体/斜体运行节点在文字替换后继续保留；
- 原格式模式会拒绝段落数量变化，避免超出保证范围后伪称无损；
- 浏览器二进制接口使用明确的 `ArrayBuffer`，通过严格TypeScript检查。

### 服务端、隐私与自动任务

- Worker API、隐私守卫、构建脚本、Service Worker 和文本 Worker 通过 Node 语法检查；
- D1迁移定义用户、邀请、会话、成员、作品修订、DOCX、榜单、备份、站点设置、批注、建议、发布记录和审计表；
- 私人内容请求先经过 `mojie-privacy-guard.mjs`；
- 云端作品列表只返回当前用户拥有有效 `work_members` 记录的作品；
- 全局 Owner/Admin 不会因全局角色自动通过普通正文接口；
- Worker构建产物包含核心API、扩展API、隐私守卫和Scheduled入口；
- 生成的Worker配置包含Cron触发器；
- PWA Service Worker 明确排除 `/api/`、导航请求和私人HTML；
- 长文本检查可由独立 Web Worker 执行。

## 本轮新增的可见功能

- 作品级一次性邀请；
- 已注册用户直接授权；
- 作品成员列表与撤权；
- 章节级和选区级批注；
- 原文/建议对比；
- 锚点核对后接受或拒绝建议；
- 结构化章节细纲与章节状态、目标、伏笔和钩子；
- Owner/Admin 管理后台；
- 用户状态、邀请记录和审计日志；
- 服务端可配置站点名称、默认邀请时长和回收站默认保留期；
- 可安装 PWA；
- 私密安全的静态资源缓存；
- 长文本后台检查。

## 仍需真实部署环境验收

以下行为需要绑定 D1、R2 和真实外部存储后验证，仓库单元测试不能替代：

1. 未登录访问云端数据返回401；
2. 无作品成员关系的全局管理员读取正文返回403；
3. viewer 写入返回403；
4. commenter 可以批注和建议但不能直接写正文；
5. 成员撤权后旧会话不能继续读取作品；
6. 同一作品旧修订写入返回409；
7. 邀请过期、撤销或使用后不能再次注册；
8. DOCX原件下载哈希与上传一致；
9. 排行榜来源重定向仍受授权域名白名单限制；
10. WebDAV、R2和S3备份可创建并在到期后删除；
11. Service Worker 的 Cache Storage 不含认证API或私人页面；
12. 审计日志覆盖登录、邀请、成员、批注、建议、正文、DOCX、榜单和备份操作。

## 仍需部署者提供的外部资源

1. Cloudflare D1 数据库ID；
2. DOCX与备份R2 Bucket；
3. `MOJIE_ADMIN_TOKEN`；
4. `MOJIE_BACKUP_MASTER_KEY`；
5. 实际WebDAV或S3兼容存储账号；
6. 起点/番茄已授权来源网址；
7. 生产域名和部署权限。

完整配置步骤见 `docs/cloud-deployment.md`。未绑定资源时，对应接口必须返回明确配置错误，不得伪装成已完成云端存储、备份或自动抓取。

## DOCX保证范围

- **完全字节无损：** 导入后未修改，直接导出原件；
- **格式结构保留编辑：** 段落数量不变时，保留原OOXML包和运行格式节点，仅替换正文文字；
- **不作虚假保证：** 任意增删段落、重排复杂文本框或重构Word对象时，不宣称与原文件逐字节相同。
